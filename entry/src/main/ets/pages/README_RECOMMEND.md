# 个性化推荐 + 自定义行程规划功能说明

## 功能概述

本功能实现了完全离线的个性化推荐和自定义行程规划系统，包括：

1. **兴趣选择页面** (`InterestSelectPage.ets`)
   - 用户可选择多个兴趣标签（8-12个）
   - 支持多选，标签以Toggle按钮形式展示
   - 选择后生成个性化推荐

2. **推荐列表页面** (`RecommendPage.ets`)
   - 根据用户选择的兴趣标签匹配藏品
   - 匹配规则：标签交集越多排名越前
   - 支持多选藏品加入行程
   - 不足10件时自动补充热门藏品

3. **自定义行程页面** (`MyItineraryPage.ets`)
   - 显示已添加的藏品列表
   - 支持拖拽排序（长按列表项）
   - 支持删除藏品
   - 显示预计游览时间（每件藏品时长总和）
   - 数据持久化保存

## 数据结构

### 兴趣标签
定义在 `common/constants/InterestTags.ets`，包含：
- 古代中国、西方绘画、雕塑、陶瓷、亲子友好
- 科技文物、革命历史、玉器、书法、青铜器
- 文艺复兴、印象派

### 藏品数据
定义在 `services/ArtifactService.ets`，包含15件示例藏品，每件包含：
- id, title, museum, desc, image
- tags（标签数组）
- duration（预计游览时长，分钟）
- isPopular（是否热门）

## 使用流程

1. **首次使用**：点击导航栏"开始探索" → 选择兴趣标签 → 生成推荐
2. **查看推荐**：推荐页面显示匹配的藏品列表
3. **添加行程**：在推荐页面多选藏品 → 点击"加入行程"
4. **管理行程**：进入"我的行程"页面 → 拖拽排序或删除

## 技术实现

- **数据持久化**：使用 `AppStorage` 存储用户兴趣和行程数据
- **推荐算法**：基于标签交集数量排序，交集为0不显示
- **拖拽排序**：使用List组件的 `onItemMove` 回调
- **完全离线**：所有数据本地打包，使用 `$rawfile` 加载图片
